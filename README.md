


# Joybs – платформа для анализа IT-рынка

**Joybs** – это веб-приложение, которое помогает студентам и начинающим IT-специалистам анализировать рынок труда, находить востребованные навыки и строить карьерный путь на основе данных о вакансиях с hh.ru.

Проект состоит из:
- **Фронтенда** – HTML/CSS/JS с использованием Chart.js и D3.js для визуализации.
- **Бэкенда** – FastAPI (Python), который предоставляет API для данных о вакансиях, аналитики и графа профессий.

---

## ⚠️ Важно: для работы необходим локальный сервер

Фронтенд **не работает без бэкенда**, так как все данные загружаются через API-запросы к серверу.  
Вы **не сможете** открыть `index.html` напрямую в браузере – нужно запустить сервер и обращаться к нему через `http://127.0.0.1:8000`.

---

## Технологии

- **Бэкенд:** Python 3.9+, FastAPI, SQLite (aiosqlite)
- **Фронтенд:** HTML5, CSS3, JavaScript (vanilla), Chart.js, D3.js
- **База данных:** SQLite (файл `hh_parser.db`)

## Установка и запуск

### 1. Клонируйте репозиторий

```bash
git clone <url-репозитория>
cd Joybs
```

### 2. Установите зависимости Python

Рекомендуется использовать виртуальное окружение:

```bash
python -m venv venv
source venv/bin/activate   # для Linux/Mac
# или
venv\Scripts\activate      # для Windows
```

Установите зависимости:

```bash
pip install -r requirements.txt
```

Если файла `requirements.txt` нет, создайте его со следующим содержимым:

```
fastapi
uvicorn
aiosqlite
```

### 3. Подготовьте базу данных

В корне проекта должен лежать файл `hh_parser.db` с данными о вакансиях.  
Если у вас его нет, вам нужно создать его и заполнить данными (например, с помощью парсера hh.ru).  
Структура таблицы `vacancies` должна соответствовать полям, используемым в API (см. код).

### 4. Запустите сервер

Из корневой директории проекта выполните:

```bash
uvicorn backend.main:app --reload
```

Сервер запустится на **http://127.0.0.1:8000**.

### 5. Откройте сайт в браузере

Перейдите по адресу **http://127.0.0.1:8000** – вы увидите главную страницу Joybs.

> **Обратите внимание:** если вы попытаетесь открыть `frontend/index.html` напрямую, API-запросы не будут работать, так как браузер блокирует кросс-доменные запросы. Всегда запускайте сервер и обращайтесь к нему.

## Как пользоваться

- **О проекте** – информация о платформе, граф сходства профессий, категории профессий.
- **Аналитика** – общая статистика, распределения, зарплатные графики, топ навыков, матрица совместной встречаемости навыков.
- **Агрегатор** – фильтрация и просмотр вакансий с возможностью поиска по тексту, навыкам, компании, городу, зарплате и т.д.

---

## Настройка контекста (фильтрация по группе/профессии)

В левой панели можно выбрать контекст:
- **Весь рынок** – данные по всем вакансиям.
- **Группа** – фильтрация по категории профессий (например, "Языки программирования").
- **Профессия** – фильтрация по конкретной профессии (например, "Python разработчик").

## Возможные проблемы и решения

| Проблема | Решение |
|----------|---------|
| Сервер не запускается | Проверьте, что вы находитесь в корневой папке проекта и установлены все зависимости. |
| API возвращает ошибку 500 | Убедитесь, что база данных `hh_parser.db` существует и содержит нужные таблицы. |
| Фронтенд не видит данные | Откройте консоль разработчика в браузере и проверьте, что запросы к `/api/...` выполняются без ошибок CORS. |
| Шрифт Science Gothic не отображается | Проверьте подключение к Google Fonts (требуется интернет). |

## Деплой на GitHub Pages (только фронтенд)

Вы можете опубликовать статические файлы фронтенда на GitHub Pages, но **API не будет работать**, так как GitHub Pages не поддерживает запуск Python-бэкенда.  
Для полноценной работы вам понадобится отдельный хост для бэкенда (например, Render, Railway или Heroku) и указать в коде фронтенда его URL вместо `/api`.
